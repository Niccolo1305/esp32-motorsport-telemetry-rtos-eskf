Zero-jitter vehicle telemetry. Open source. No compromises.

Telemetria is an open-source firmware for M5Stack Atom S3 / S3R that fuses 50 Hz IMU with 10 Hz GPS through a 5D/6D Error-State Kalman Filter. Logs 256-byte records to SD, streams JSON over MQTT, exports to MoTeC i2 Pro. Built for motorsport by people who got tired of black-box data loggers.

§ 02 · Pipeline

The math, end-to-end. Sixteen steps from chip to SD.

// Task_Filter — Core 1 · priority 2 · 50 Hz · FreeRTOS
STEP 0   Dynamic dt from esp_timer_get_time()
STEP 1   Axis convention remap (chip → pipeline)
STEP 2   Ellipsoidal calibration    a_cal = W · (a_raw − b)
STEP 3   Geometric alignment        rotate_3d(a_cal, φ, θ)
STEP 4   Mounting bias residuals    a_r = a_r_raw − bias_a
STEP 5   Gyro bias + rotation       g_r = rotate_3d(g − bias_g)
STEP 6   VGPL kinematic compensation (centripetal + longitudinal)
STEP 7   Madgwick AHRS              dual-gate adaptive β
STEP 8   Gravity cancellation       lin_a = a_r − grav(q)
STEP 9   Statistical engine · RAW variance buffer (50 samples)
STEP 10  GPS snapshot               gps_mutex · 2 ms timeout
STEP 11  Stationary detection       var + sog + mean_gz gates
STEP 12  ZARU update                static + straight-line
STEP 13  ESKF 5D predict            X = [px py vx vy θ]
STEP 14  NHC v_lateral = 0          R_nhc = 0.5 (m/s)²
STEP 15  ESKF correct · Joseph form 3-stage sequential
STEP 16  End-of-pipe EMA            α = 0.06 · τ ≈ 313 ms
             → sd_queue → Task_SD_Writer (256 B, aligned)
§ 03 · Why
01

FreeRTOS multi-core

Task_I2C on Core 0 at priority 3. Task_Filter on Core 1 at priority 2. Zero jitter by design: mailbox queue (depth 1) with xQueueOverwrite guarantees freshest data with no blocking.

02

5D / 6D shadow ESKF

Production 5D filter drives display, MQTT, logs. The 6D shadow runs the same data with an augmented gyro-bias state. Offline A/B testing, zero runtime cost. Both use Joseph-form covariance updates.

03

Sector-aligned SD

256 bytes per record, 2 records per SD sector. Cross-sector partial-write misalignment can't happen. bin_to_csv.py resyncs automatically if it ever does.

§ 04 · Changelog

Every version, every fix, in plain English.

v1.7.6WORKING · Oct 2026

Sector-aligned records + SD resync. TelemetryRecord padded from 242 B to 256 B with reserved[14]. 2 records = 1 SD sector (512 B), eliminating cross-sector partial-write misalignment. bin_to_csv.py gains automatic resync: scans up to 4096 bytes for the next valid alignment.

v1.7.5SD hotfix

Removes File.position() / ftell() from the runtime recovery path. A transient invalid position result could previously kill Task_SD_Writer even when File.write() had not actually failed. Progress now trusts only write() return value.

v1.6.0HAL

AtomS3R Bosch-direct HAL introduction. Bmi270Provider for explicit bring-up on M5.In_I2C. M5Unified demoted to board/display/power only. Production IMU path no longer owned by M5.

v1.2.0G-sensitivity

3×3 K_gs matrix measured by differential 6-face method. Corrects MPU-6886 gyro response to linear acceleration. Matters most on gz (the ESKF heading axis), where the effect survived boot calibration.

v0.9.3Spiral fix

3-stage sequential Kalman update. Position alone wasn't enough — P(pos,vel) cross-covariance was too small to transfer information to velocity, so kf_vel underestimated by ~30 % and heading drifted into a spiral. Adding scalar speed and Course-Over-Ground updates closed the loop.

v0.1.0baseline

Full port from MicroPython to C++ Arduino/ESP-IDF. MPU-6886 IMU via M5Unified. Basic pipeline: read → EMA filter → MQTT JSON at 10 Hz. Humble beginnings.

§ 05 · Install

Flash it, mount an SD, drop in a wifi config.

$ git clone https://github.com/<you>/telemetria.git
$ cd telemetria
$ arduino-cli compile --fqbn m5stack:esp32:m5stack_atoms3
$ arduino-cli upload   --fqbn m5stack:esp32:m5stack_atoms3 -p /dev/ttyACM0

# On the SD card, root:
$ cat > /Volumes/TEL/wifi_config.txt <<EOF
SSID=garage-wifi
PASSWORD=hotlap2026
MQTT_BROKER=192.168.1.10
MQTT_PORT=1883
MQTT_TOPIC=telemetry/car42/data
EOF

# Launch the live dashboard:
$ python tools/dashboard.py --broker 192.168.1.10
